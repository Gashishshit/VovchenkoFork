# Практическое задание: Система детекции рудозасорения на конвейерных лентах

---

## 1. Введение и контекст

**Рудозасорение** — это присутствие породных включений (мусор, арматура, крупные камни) в рудном потоке на конвейерной ленте. Своевременное обнаружение засорения критично для:

- предотвращения перегрузки и износа оборудования;
- снижения затрат на переработку.

В рамках задания предлагается разработать систему детекции объектов на основе трансформеров, работающих с визуальной информацией (DETR/ViT/Dino или др.), способную в режиме реального времени обнаруживать засорение по видеозаписям с промышленных камер.

---

## 2. Датасет

Датасет предоставляется отдельно в сообщении и **не входит в состав репозитория**. Датасет представлен в виде «сломанного» формата YOLO. 

### 2.2 Классы объектов

- `0` — арматура/засорение
- `1` — большие камни
- фон и руда не размечены

### 2.3 Особенности данных

- Изображения получены с камер разного качества.
- Условия съёмки варьируются: разное освещение, запылённость, угол обзора.
- Классовый дисбаланс: некоторые классы встречаются значительно реже.
- Некоторые виды рудозасорения визуально схожи с рудой.
- Часть данных содержит motion blur из-за движения ленты.

---

## 3. Задача

Разработать пайплайн детекции рудозасорения, включающий:

1. **Предобработку** входных данных (нормализация, аугментации с учётом промышленной специфики).
2. **Модель детекции** на основе DETR или другого визуального трансформера.
3. **Постобработку** предсказаний (фильтрация, NMS или equivalent для DETR-like моделей).
4. **Инференс** на тестовых снимках с визуализацией результатов.

## 5. Метрики оценки

### 5.1 Ключевая метрика — FPR (False Positive Rate)

FPR = FP/(FP + TN), по количеству снимков

Минимизация FPR является **приоритетной задачей**: ложные срабатывания (детекция засорения там, где его нет) приводят к необоснованным остановкам конвейера и производственным потерям.

> ⚠️ **Важно:** снижение FPR не должно достигаться за счёт тривиального решения «никогда не детектировать» — дополнительно контролируется Recall.


## 7. Рекомендации по снижению FPR

При разработке системы рекомендуется рассмотреть следующие подходы:

- **Порог уверенности (confidence threshold):** подбор порога на валидационной выборке с акцентом на минимизацию FP.
- **Temporal consistency:** для последовательных кадров — фильтрация детекций, которые не подтверждаются в соседних кадрах.
- **Hard Negative Mining:** специальное обучение на трудных отрицательных примерах.
- **Class-weighted loss:** увеличение штрафа за FP относительно FN в функции потерь.
- **Test-Time Augmentation (TTA):** усреднение предсказаний по нескольким аугментациям.
